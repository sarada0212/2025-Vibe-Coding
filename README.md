2026 Vibe Coding 期末專案：簡易匯率計算器 (Currency Converter)
1. 專題願景 (Design Philosophy & Vibe)

* **設計理念：** 打造一個簡單、直觀的匯率轉換工具，創造出專業且現代的金融氛圍。就像是一個便捷的線上錢包助手，讓人感覺安全、效率，並帶點科技感。
* **視覺風格 (Vibe)：**
    * **色調：** 採用**藍色調**為主，象徵信任與穩定。
    * **視覺焦點：** 使用抽象圖表和貨幣符號元素，**避免使用人物圖片**，讓使用者能更專注於數字與功能。
* **目標對象：** 適合初學者或日常使用者，能快速計算常見貨幣（如 TWD, USD, JPY 等）。
* **技術策略：** 強調靜態數據的可靠性，捨棄複雜的 API 即時同步，確保網頁載入速度快且無需維護後端，專注於前端介面的流暢度。

2. Gemini 協作紀錄 (Prompt Log)

在開發過程中，我使用「Vibe Coding」的方式與 AI 溝通。以下是引導專案方向最關鍵的一個指令：

 核心 Prompt：**
> 「重新做一個 新的主題 匯率計算 匯率用當前的就好不用世界同步。」
> *(這個指令成功避免了串接實時 API 的複雜度，讓開發能專注在介面與使用者體驗上，快速推進專案進度。)*

其他修正指令包含：
* **針對圖片：** 「網頁的圖片換一下 不要用人物」
* **針對手機版功能：** 「script.js 不要用到」（這讓 AI 改用 CSS Checkbox Hack 來實作漢堡選單）

3. 問題解決紀錄 (Problem Solving Log)

在與 AI 協作開發的過程中，主要遇到了顯示與素材的問題。以下是我的解決過程：

| 問題點 | 遇到的困難 | Gemini 的解決方案與我的提示 |
| :--- | :--- | :--- |
| **1. 頁面空白** | 初始版本使用了 `scroll-reveal` 動畫，導致內容預設 `opacity: 0`，在沒有載入 JS 的情況下網頁變成一片白。 | **提示詞：**「空白的」（附上截圖）<br>**解決：** AI 診斷出是動畫問題，移除了隱藏效果，將所有內容設為預設可見 (`opacity: 1`)，讓頁面立即恢復正常。 |
| **2. 圖片模糊/失效** | 原始生成的 Unsplash/Getty 連結失效，或解析度過低（太糊），且部分圖片包含我不想要的人物元素。 | **提示詞：**「第二張圖片 跑不出來 而且圖片太糊」<br>**解決：** AI 搜尋並替換為更清晰的抽象財經圖片（如 iStock 的錢幣插圖），並加入了背景色作為圖片載入失敗時的備案。 |
| **3. 手機選單卡住** | 在決定移除 `script.js` 以簡化專案後，手機版的漢堡選單無法運作。 | **提示詞：**「script.js 不要用到」<br>**解決：** AI 改寫了 HTML 與 CSS，利用 `<input type="checkbox">` 的技巧來控制選單開關，成功在不寫 JS 的情況下達成 RWD 功能。 |

4. 檔案結構說明

本專案採用極簡架構，無需後端即可運作：

* `index.html`：網頁主結構（包含匯率計算介面、匯率表、關於我們）。
* `style.css`：視覺美化（RWD 響應式設計、藍色系主題配色、CSS 選單互動）。
* *(本專案不依賴 script.js，所有互動效果皆由 CSS 完成)*

5. 自我心得與反思

透過這次 Vibe Coding 的實作，我深刻感受到 AI 輔助開發的強大與便利：

1.  **開發效率極高：** 原本以為從零開始切換主題到「匯率計算器」會很複雜，但透過簡單的自然語言提示，AI 就能迅速生成完整的 HTML/CSS 框架。
2.  **迭代的重要性：** AI 並非一次完美，例如圖片載入失敗或動畫導致的空白問題，都需要透過「人」的判斷去指引 AI 修正。
3.  **成就感：** 雖然過程中遇到 Bug，但看著作品從概念到上線只花了短時間，且成品具備專業感，這激勵了我未來更願意嘗試使用 AI 工具來實現創意。
